### 1. 完了通知の仕様について — ご回答の訂正

前回お送りした 4 項目のうち、①「自動送信される対象」について、こちらの説明が不正確でした。改めて整理いたしますと:

- **「完了通知」の自動配信レーン** = **全コース 100% 完了の方のみ・お一人につき 1 度だけ** 送信
- 受講途中の方 (進捗率 20%, 50% 等) には、このレーンからは **何も送信されません**
- 前回お送りしたスクリーンショット (受講状況 → Gmail 下書き) で表示される **「途中経過の進捗レポート (進捗率 20% 等)」は、現状では手動で 1 件ずつ下書き作成いただく機能** で、完了通知の自動配信とは別物です

XX 様が前回いただいたご返信から、「途中の受講者へも定期的に自動配信してほしい」というご要望と拝察いたしました。これは、当初の設計では Phase 3 候補として先送りされていた機能 (Issue #346) ですが、改めて開発判断として **Phase 3「進捗レポート定期自動配信」として正式実装することを決定**し、すでに本番環境へのデプロイまで完了いたしました。あわせて続く Phase 4 で「配信前プレビュー機能」も追加実装済みです。順にご案内いたします。

### 2. Phase 3 完成: 進捗レポート定期自動配信

受講途中の方を対象に、定期的に進捗レポートを Gmail 下書き経由で自動配信する機能を新規実装いたしました。

| 項目 | 仕様 |
|------|------|
| **配信対象** | 受講中の全受講者 (100% 完了の方は別途完了通知レーンが対応) |
| **配信頻度** | 業務スーパー管理者が指定された曜日 / 時刻に定期実行 |
| **メール本文** | 進捗率・受講期限・推奨ペースを含む進捗レポート (既存の手動下書き機能と同じテンプレート) |
| **CC** | テナントオーナー様 + 設定された追加 CC アドレス |
| **テナント単位の opt-in** | テナントごとに「進捗レポート定期配信を有効化」スイッチで個別 ON/OFF 可能 (既定は OFF) |

「完了通知」と「進捗レポート」は **独立した 2 つの配信レーン** として動作し、両方有効・片方のみ有効・両方無効、いずれの組合せも可能です。

### 3. Phase 4 完成: 配信プレビュー機能

設定画面で「次回の送信対象一覧」「送信予定のメール本文プレビュー」をボタン 1 つで確認できる機能を追加いたしました。**実際の送信は行わず読み取りのみ** ですので、安心して何度でもお試しいただけます。

両レーンとも同じ操作で確認できます:

1. スーパー管理画面右上のメニューから **「配信設定」** を開く (URL: `/super/dispatch-settings`)
2. 画面下部の以下のセクションで「プレビューを取得」ボタンを押下:
   - **「完了通知 配信プレビュー」** — 次回の完了通知で送信される方の一覧 + メール本文
   - **「進捗レポート 配信プレビュー」** — 次回の進捗レポートで送信される方の一覧 + メール本文
3. 表示される項目:
   - 送信対象テナント / 受講者一覧
   - スキップ理由内訳 (期限超過 / 既送信済等)
   - 送信予定メールの件名 + 本文プレビュー
   - (進捗レポートのみ) 推定処理時間 + 300 名超の場合の警告

### 4. 本番稼働開始までの段取り

下記の手順を XX 様ご自身の画面操作で順に進めていただくことで本番稼働が開始します。**最終的な ON 操作は AI も開発側も代行せず、XX 様の手元** で実施いただく設計としております (誤配信防止 / 監査責任の所在明確化のため)。

#### 進捗レポートを稼働開始する場合

| Step | 操作 | 画面 |
|------|------|------|
| 1 | **対象テナント** の管理画面で「このテナントへの進捗レポート定期配信」スイッチを **ON** (テナントごと個別に) | テナント詳細ページ |
| 2 | **配信曜日 / 時刻** を設定 | `/super/dispatch-settings` の「進捗レポート 定期配信」セクション |
| 3 | **「プレビューを取得」** で対象者・本文を確認 | 同上 「進捗レポート 配信プレビュー」 |
| 4 | 全社共通 **「進捗レポート定期配信を有効化」** スイッチを **ON** (マスタートグル) | 同上 |

> ※ Step 4 の ON を行うまでは Step 1-3 を完了していても **何も送信されません** (システム側で no-op に設計されています)。安心して事前準備が可能です。

#### 完了通知を稼働開始する場合

| Step | 操作 | 画面 |
|------|------|------|
| 1 | **配信曜日 / 時刻** を確認 (既に設定されている場合は確認のみ) | `/super/dispatch-settings` の「完了通知 定期配信」セクション |
| 2 | **「プレビューを取得」** で対象者・本文を確認 | 同上 「完了通知 配信プレビュー」 |
| 3 | 画面最上部の **「配信を有効化」** スイッチを **ON** (マスタートグル) | 同上 |

> ※ Step 3 の ON を行うまでは何も送信されません。

両レーンとも **マスタートグルを再度 OFF にすれば即座に配信停止** されますので、稼働後の運用上の不都合が出た場合の緊急停止も画面操作のみで完結いたします。
