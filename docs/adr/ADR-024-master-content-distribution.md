# ADR-024: マスターコンテンツ配信機能

## ステータス
採用

## コンテキスト
スーパー管理者が統一的なコンテンツ（コース・講座・動画・テスト）を作成し、各テナントに一括配信する機能が必要。

## 決定事項

### コピー型配信
- マスターコースを対象テナントにディープコピーする
- コピー後はテナント側で自由に編集可能（マスターとの同期なし）
- マスター更新は既存コピーに影響しない

### `_master` 疑似テナント
- 既存の `FirestoreDataSource` をそのまま再利用
- Firestoreパス: `tenants/_master/courses/...`
- `RESERVED_TENANT_IDS` に追加し一般ユーザーの登録を防止

### コース単位配信
- 選択したコースの講座・動画・テストを丸ごとディープコピー
- 新しいドキュメントIDを生成、FK参照（courseId, lessonId, lessonOrder）をリマップ
- `sourceMasterCourseId` と `copiedAt` でコピー元を追跡
- 重複チェック: 同一 `sourceMasterCourseId` が既存なら skip

### GCS動画の共有
- メタデータのみコピー、GCSファイルは共有（同じ `gcsPath` を参照）
- ストレージコストを削減

### Firestoreバッチ書き込み
- 500件上限でチャンキング（Firestore batch制限）

## 影響
- `Course` 型に `sourceMasterCourseId?` と `copiedAt?` を追加
- `toCourse` マッパーで新フィールドを読み取り
- スーパー管理者ルートに `/master/*` エンドポイントを追加

## 代替案
- **同期型**: マスター更新がテナントに自動反映 → 複雑性が高く、テナント独自編集と競合
- **テンプレート型**: コース構造のみコピー → コンテンツ（動画・テスト）も含めたいため不採用
